Используй агент benchmark-analyzer для анализа: $ARGUMENTS

Если файл/модуль не указан — найди последние отчёты в Results/Profiler/ и предложи.

Алгоритм:
1. Найди файл(ы) в Results/Profiler/ по имени: $ARGUMENTS
   - Ищи .md и .json варианты
   - Если несколько — берй последний по дате в имени
2. Прочитай отчёт и построй timeline операций
3. Найди узкие места (HtoD overhead, долгие kernels, лишние sync)
4. Сравни если есть несколько отчётов одного модуля (regression?)
5. Выдай конкретные рекомендации с файл:строка

Формат: таблица timeline + список проблем + рекомендации с ожидаемым эффектом.
