# 📨 MemoryBank/prompts/

> Готовые промпты для копипасты в новый чат Claude Code.
> Удобно когда работа растянута на несколько дней / нужно железо / ждём решения.

---

## 🗂️ Структура

```
prompts/
├── README.md                                ← этот файл
├── TEMPLATE_continue_review.md              ← шаблон для любого follow-up ревью
├── TEMPLATE_new_review.md                   ← шаблон "начать новое ревью"
└── {YYYY-MM-DD}_{topic}.md                  ← конкретные промпты с датой
```

---

## 🚀 Как использовать

### Вариант A — простой копипаст (рекомендую)
1. Открыть нужный `.md` файл
2. Скопировать содержимое блока ``` ``` (или всё тело промпта)
3. Вставить в новый чат с Кодо

### Вариант B — через команду Claude Code
Если новая сессия уже открыта в этом workspace, просто скажи:
> «Выполни промпт из `MemoryBank/prompts/2026-04-16_continue_core_spectrum.md`»

Кодо сам прочитает файл и подхватит контекст.

### Вариант C — через шаблон (для НОВЫХ ревью)
1. Скопировать `TEMPLATE_continue_review.md` → `{YYYY-MM-DD}_{topic}.md`
2. Заменить переменные `{DATE}`, `{REVIEW_NAME}`, `{TASK_FILE}`, `{CHANGELOG_FILE}`
3. Сохранить, скопипастить в чат

---

## 📋 Зачем это нужно

- **Сохраняет контекст между сессиями** — новая сессия Кодо не помнит вчерашний чат, но помнит MemoryBank
- **Экономит токены** — один короткий промпт вместо многостраничного резюме
- **Воспроизводимость** — любой (Alex / напарник / будущая Кодо) может продолжить работу
- **Для ревью, которые ждут железа / OK Alex** — можно "заморозить" задачу и вернуться через недели

---

## 📜 Правила при создании промпта

1. **Короткий** — 15-25 строк максимум (не раздувать — Кодо сама прочитает нужные файлы)
2. **Ссылки на MemoryBank** — вместо копипаста содержимого
3. **Явные границы задачи** — что делать / что НЕ делать
4. **Красные линии** — CMake, git push, tag — всегда с `⚠️`
5. **Формат**: обёрнутый в ``` ``` тройной бэктик блок кода (чтобы удобно копировать)
