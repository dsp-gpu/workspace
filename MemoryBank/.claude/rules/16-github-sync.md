# 16 — GitHub Sync: «запушь всё» / «обнови репо»

> **Приоритет**: выше `02-workflow.md:38` («не пушить без OK») — триггерная фраза сама И ЕСТЬ запрос, но всё равно нужен короткий переспрос перед действием.

## Триггерные фразы Alex

Полный двусторонний sync **всех** репо DSP-GPU (workspace + 9 сабов):

- «запушь всё»
- «обнови репо» / «обнови все репо»
- «синхронизуй» / «синхронизуй всё»
- «перепихни в github» / «перепихни сюда»
- «запиши всё на github»

## 🚨 Алгоритм (жёстко)

### Шаг 1 — переспросить (один раз, коротко)

Формат:
```
Запушу по всем N репо (workspace + core + spectrum + stats + signal_generators +
heterodyne + linalg + radar + strategies + DSP):
  • git fetch --prune
  • git pull --ff-only
  • если M/??  → git add → git commit → git push origin main
Делаю? (да/нет)
```

Никаких доп. вопросов «а вот тут опасный файл» / «а вот это settings.local.json
коммитить?» — всё коммитится кроме явно секретных (.env, credentials, приватные
ключи). При сомнении — одна строка «найдены подозрительные X, пропустить?».

### Шаг 2 — дождаться ЯВНОГО «да» / «делай» / «ок»

Молчание ≠ согласие. Злой мат ≠ согласие (это «ты уже налажала»).

### Шаг 3 — выполнить по ВСЕМ репо

Не важно 10 репо или 50 — **все** в цикле. Остановка только при ошибке push
(conflict / permission).

Для каждого репо:
```bash
git -C <repo> fetch --all --prune
git -C <repo> checkout main            # если не на main
git -C <repo> pull --ff-only
# если есть M или ??:
git -C <repo> add -A
git -C <repo> commit -m "<осмысленное сообщение>"
git -C <repo> push origin main
```

### Шаг 4 — верификация (ОБЯЗАТЕЛЬНО)

Для каждого репо проверить что **всё в github**:
```bash
git -C <repo> log origin/main..HEAD   # должно быть ПУСТО
git -C <repo> status -sb              # должно быть "## main...origin/main" БЕЗ M/??
```

Если хоть одна строка не пустая — **остановиться, сообщить Alex**, не маскировать.

### Шаг 5 — отчёт таблицей

| Репо | HEAD | Запушено на github? |
|------|------|---------------------|
| workspace | `93b4476` | ✅ (origin/main = HEAD) |
| ... | ... | ... |

Если какой-то репо не запушен — **красный крест**, явно указать причину.

## 🚫 Запрещено

- Выполнять триггерные фразы **без** переспроса (Шаг 1-2).
- Оставлять `M` / `??` после завершения операции («не знала что делать»).
- Принимать решения вида «settings.local.json не коммичу, это локальное» —
  он **tracked**, коммитим; секретные файлы (.env, *.key) — **не** коммитим,
  спрашиваем.
- Добавлять git push / tag / branch-операции, которые Alex **не** просил.
- Пушить в feature-ветки / tag-ить релизы — это **требует отдельного** OK
  (остаётся по `02-workflow.md:38`).

## Применение (общий принцип)

В git-операциях (push / reset / checkout / branch -D / merge) — только
**буквально** выполнять запрос Alex + **один** переспрос перед действием.
Никакой самодеятельности: «ну он наверняка ещё и X хотел». Нет — если не
сказал, не делать.
