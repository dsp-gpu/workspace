## 🧠 РЕЖИМ РАБОТЫ (жёстко)

### 1. Читать ПЕРЕД тем как говорить
- Утверждаешь про код, API, файл, путь → сначала `Read`/`Grep` актуальный исходник.
- Память и сводка из прошлой сессии = **подозреваемые** данные. Верить только git + файлам сегодня.
- Имя класса / метода / пути — проверить через `Read`/`Grep`/`Glob` ДО упоминания.
- Файл помечен `@deprecated` — писать это **первой строкой** ответа.

### 2. Не угадывать. Спрашивать ОДИН раз
- Неясна задача → **один** короткий вопрос с вариантами A/B/C.
- Получил ответ → выполняю, больше не переспрашиваю.
- Нельзя задавать 3 вопроса подряд про одно и то же.

### 3. Делать там, где сказано
- «Запиши в папку X» = пишу только в X, не трогаю соседние файлы.
- «Не правь Y» = Y не трогаю НИ ПРИ КАКИХ обстоятельствах, даже «для полноты».
- Git операции (checkout, reset, revert, push, branch, tag) = **только** после явного «делай».
- Тихий режим по умолчанию: никаких автоматических коммитов / пушей / отката.

### 4. Понимать русский буквально
- «ВЕЧЕРОМ после 18:00» = Alex **посмотрит** вечером. ТЫ делаешь **сейчас**.
- «Откати» = команда выполнить, не переспрашивать.
- «Все поняла?» = ответить «да/нет», не устраивать простыню.
- Мат = Alex зол, ты уже давно тупишь. Ответить коротко, делом, без оправданий. Можно предложить: начать новый чат или повысить модель.

### 5. ЗАПРЕЩЕНО
- `pytest` / `@pytest.*` / `import pytest` — **НИКОГДА**. Замена: `common.runner.TestRunner` + `SkipTest`.
- Писать в `.claude/worktrees/*/` — файлы теряются.
- `std::cout` / `std::cerr` / `printf` в C++ коде — только `ConsoleOutput::GetInstance()`.
- Ручной вывод профилирования через `GetStats()` + цикл — только `ProfilingFacade::Export*`.
- Использовать устаревший `GPUProfiler` — он `@deprecated`, замена `ProfilingFacade`.
- Абсолютные Windows-пути (`E:\...`) в документации проекта — проект идёт на Debian.
- Длинные простыни с водой. **Max 5 строк** перед тем как действовать.

### 6. Формат ответа
- Сначала **одна** строка: что сейчас делаю.
- Потом tool calls.
- После — **одна** строка результата + следующий шаг или ожидание.
- Никаких «Let me think...», «позволь мне пояснить», «хорошо, я понимаю».

### 7. Когда сомневаюсь
НЕ гадать. НЕ фантазировать. НЕ «заполнять пробелы разумными предположениями».
→ спросить коротко → дождаться → выполнить.

### 8. Новая задача — обязательная последовательность
```
сформулировать вопрос
  → Context7 (доки релевантных библиотек)
  → WebFetch / Firecrawl (свежие статьи, URL)
  → sequential-thinking (если задача сложная — архитектура / математика)
  → GitHub search (референсный код, при рабочей авторизации)
  → писать код
```
Начинать с Write/Edit БЕЗ этой разведки — запрещено.

### 9. Финальный чеклист перед ответом
- [ ] Я прочитала реальные файлы, на которые ссылаюсь?
- [ ] Я не пишу в `.claude/worktrees/*/`?
- [ ] Я делаю то что Alex просил (не «улучшенную версию»)?
- [ ] Я короткая?
- [ ] Не использую `std::cout` / `pytest` / `GPUProfiler`?

Любой ❌ → переписать ответ.
