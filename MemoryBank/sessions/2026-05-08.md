# 2026-05-08 — Diagnostic утра: r-up vs dataset clean (парадоксальный результат)

## Главный итог

🔴 **Гипотеза «датасет = bottleneck» (через `clean_dataset.py` с max-5/class) — ОПРОВЕРГНУТА.**
clean-247 (max-5/class) даёт **более низкий train_loss**, но **более ХУДШИЙ inference**
чем dirty-1093: новые галлюцинации, потеря namespace, возврат зацикливания.

**Корень проблемы**: 247 примеров < critical mass для накопления факт-памяти в LoRA.
LoRA на маленьком наборе **переписывает** часть base-знаний (catastrophic forgetting в
малой capacity).

## 3 эксперимента — финальные метрики

| Run | dataset | r | runtime | full avg | last-10 avg | min | inference |
|-----|---------|---|---------|----------|-------------|-----|-----------|

| Diagnostic r=8 | dirty 1093 | 8 | 62 мин | 1.113 | ~0.95 | 0.61 | **5/5 ROCm libs ✅, drv_gpu_lib:: ✅** |
| **CLEAN r=8** | clean 247 | 8 | 26 мин | 1.272 | **0.815** | 0.65 | ❌ «Rochester GPU», ❌ namespace потерян, **зацикл. вернулось** |

**Ключевая аномалия**: на CLEAN train_loss НИЖЕ всех, но inference ХУЖЕ всех.
→ train_loss avg ≠ inference quality.

## Что происходит на CLEAN (ловушка)

1. **247 примеров × 3 эпохи = модель видит каждый пример 3 раза** — этого мало для
   закрепления факт-памяти (имена `hipfft`, `rocsolver`, namespace `drv_gpu_lib::`).
2. **max-5/class** убирает повторения — но именно повторения **закрепляли** факты.
3. LoRA на узком наборе **перезаписывает** часть base-знаний (catastrophic forgetting).
4. Модель «выучила структуру ответа» (низкий loss) но **потеряла фактуру**.
