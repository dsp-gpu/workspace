### Phase 3: 🐛 Major finding — баг `chunker_cpp.py`

При verify обнаружено что у `BufferSet`, `GPUBuffer`, `HIPBuffer`, `AsyncServiceBase`, `ExternalCLBufferAdapter`, `InputData` **в исходниках brief был, но в БД `doxy_brief = null`**.

**Корневая причина** — `dsp_assistant/indexer/chunker_cpp.py:65-76`:

```python
def _get_doxy_above(node, source):
    prev = node.prev_sibling
    if prev is None or prev.type != "comment":
        return None  # ← для template-классов prev = template params!
```

Для `template<typename T> class Foo` tree-sitter-cpp выдаёт:
```
template_declaration
  ├── parameters       ← node.prev_sibling указывает СЮДА
  └── class_specifier  ← наш `node`
```

`_get_doxy_above` видит template-параметры (не comment) → возвращает `None` → **9+ ключевых template-классов теряют doxygen в БД**.

**Это была корневая причина** парадокса Long-v6 + P2 GROUNDED = 59.3% halluc: модель видит «(описание отсутствует)» для центральных шаблонов (`BufferSet`, `GPUBuffer`, `InputData`) и дополняет пустое место выдумкой.

### Phase 4: fix + reindex

Fix (15 строк):
```python
prev = node.prev_sibling
if (prev is None or prev.type != "comment") and \
   node.parent is not None and node.parent.type == "template_declaration":
    prev = node.parent.prev_sibling
```

Reindex полный `dsp-asst index build --root e:/DSP-GPU --lang cpp --force` → **531 файл, 4444 символа**.

Verify через psycopg: **21/22 классов с brief в БД** ✅
(только `is_cl_mem` под `#ifdef CL_VERSION_1_0` — indexer пропускает preprocessor-блок).
