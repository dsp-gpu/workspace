## 🖥️ Deploy на Ubuntu 24 server (через SSH)

### 1. На сервере (после установки Ubuntu 24)
```bash
# ROCm 7.2+ (AMD official) — ~5 GB
wget https://repo.radeon.com/amdgpu-install/7.2/ubuntu/jammy/amdgpu-install_7.2.60200-1_all.deb
sudo dpkg -i amdgpu-install*.deb
sudo amdgpu-install -y --usecase=rocm

# Базовые tools
sudo apt install -y build-essential cmake git python3.12-venv libcurl4-openssl-dev postgresql-15 postgresql-contrib pgvector
sudo systemctl enable postgresql --now
```

### 2. С твоего PC — rsync артефактов

```bash
SERVER="user@server-ip"

# llama.cpp source + builds (исходники, не GGUFы)
rsync -av --exclude='*.gguf' --exclude='build/' /home/alex/llama.cpp/ $SERVER:~/llama.cpp/

# GGUFы — ~80 GB суммарно (MTP 22 + 30B 18 + 14B 9 + 35B-Q8 37)
rsync -av --progress /home/alex/llama.cpp/models/ $SERVER:~/llama.cpp/models/
# Через дисковый перенос быстрее — сюзий жёсткий или USB-NVMe

# HF cache (model artifacts + tokenizers)
rsync -av /home/alex/.cache/huggingface/ $SERVER:~/.cache/huggingface/

# finetune-env (venv, dataset, scripts)
rsync -av /home/alex/finetune-env/ $SERVER:~/finetune-env/
# ВНИМАНИЕ: venv заточен под текущий Python — лучше пересоздать на сервере

# DSP-GPU MemoryBank (specs, sessions, tasks)
rsync -av /home/alex/DSP-GPU/ $SERVER:~/DSP-GPU/

# dsp-asst service
rsync -av /home/alex/dsp-asst-deploy/ $SERVER:~/dsp-asst-deploy/  # если есть отдельная папка
```

### 3. На сервере — pересборка llama.cpp

```bash
cd ~/llama.cpp
rm -rf build
cmake -B build \
    -DGGML_HIP=ON \
    -DAMDGPU_TARGETS="gfx1201;gfx908" \
    -DCMAKE_BUILD_TYPE=Release \
    -DGGML_HIP_GRAPHS=ON \
    -DLLAMA_CURL=ON
cmake --build build --config Release -j$(nproc) --target llama-server llama-cli llama-quantize
```

### 4. Postgres — миграция БД

```bash
# С PC:
sudo -u postgres pg_dump gpu_rag_dsp > /tmp/gpu_rag_dsp.sql
rsync /tmp/gpu_rag_dsp.sql $SERVER:/tmp/

# На сервере:
sudo -u postgres createdb gpu_rag_dsp
sudo -u postgres psql gpu_rag_dsp -c "CREATE EXTENSION IF NOT EXISTS vector;"
sudo -u postgres psql gpu_rag_dsp < /tmp/gpu_rag_dsp.sql

# Создать user
sudo -u postgres psql -c "CREATE USER dsp_asst WITH PASSWORD '1';"
sudo -u postgres psql -c "GRANT ALL ON DATABASE gpu_rag_dsp TO dsp_asst;"
```

### 5. SSH tunnel с ноутбука → сервер

```bash
# На ноутбуке:
ssh -L 8080:127.0.0.1:8080 -L 11434:127.0.0.1:11434 -L 7821:127.0.0.1:7821 $SERVER

# Теперь VSCode Continue на localhost:8080 дёргает llama-server на сервере
# Ollama → server:11434, dsp-asst → server:7821
```

### 6. systemd на сервере для llama-server

`/etc/systemd/system/llama-server.service`:
```ini
[Unit]
Description=llama.cpp server
After=network.target

[Service]
User=alex
ExecStart=/home/alex/llama.cpp/build/bin/llama-server \
    -m /home/alex/llama.cpp/models/qwen3.6-mtp.gguf \
    -c 4096 -fa on --reasoning off \
    --spec-type draft-mtp --spec-draft-n-max 2 \
    --host 127.0.0.1 --port 8080
Restart=on-failure

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now llama-server
```

## 🚨 Известные грабли (KEEP IN MIND)

1. **bnb 0.49.2** на AMD имеет NaN bug, но **работает** с autoresume. **bnb 0.50.0.dev0** даёт **детерминированный crash на step 4**. Не апгрейдить bnb без необходимости.
2. **Unsloth на gfx1201** не даёт обещанной −60-80% VRAM экономии. На CUDA работает, на ROCm нет.
3. **VRAM 16 GB на RX 9070** — впритык для 14B FT. 30B/35B FT — невозможны локально.
4. **Старый 35B-Q8 GGUF в Ollama blob** несовместим с свежим llama.cpp (`qwen35moe.rope.dimension_sections expected 4 got 3`). Используй **свежий** что скачали 2026-05-28.
5. **VRAM не освобождается** после крахов на ROCm в low-power state. После 3+ OOM подряд — reboot.
6. **dsp-asst в systemd `Restart=always`** — сам перезапускается. Перед train: `sudo systemctl stop dsp-asst`. После: `sudo systemctl start dsp-asst`.
7. **Continue VSCode + RAG** — Continue умеет http context provider, поэтому dsp-asst (`/search` endpoint) подключается через `contextProviders`.
