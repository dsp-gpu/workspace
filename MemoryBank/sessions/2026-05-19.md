### 5. B4 разбор FAIL'ов — точечные правки
- **`t_heterodyne_comparison.py`** — root cause найден через глубокое ревью кода:
  - Симптом: GPU vs theory off by **66 kHz** константно на всех 5 антеннах
  - Арифметика: 66 kHz = ровно 44 bins (fbin=1500 Hz) = 22 μs (μ=3 GHz/s)
  - Причина: **строка 161** — `ref_single = np.exp(1j * phase_ref)` была без conj, тогда как kernel ожидает `ref = conj(s_tx)` (см. `heterodyne_kernels_rocm.hpp:54`)
  - Kernel вычисляет `conj(rx * ref) = conj(s_rx · conj(s_tx)) = conj(s_rx) · s_tx` → +μ·τ. Без conj в ref получалось `conj(rx · s_tx)` → фаза с противоположным знаком → spectrum смещён
  - **Фикс**: `np.exp(1j * phase_ref)` → `np.exp(-1j * phase_ref)` (одна строка)
  - Результат: max df 66300 → **300 Hz** (улучшение ×220), VERDICT PASSED ✅
- **4 sub-репо t_*.py** (spectrum/linalg/radar/strategies python/) — IMPORT_FAIL → PASS через добавление `DSP/Python/` в sys.path + `GPULoader.setup_path()`
