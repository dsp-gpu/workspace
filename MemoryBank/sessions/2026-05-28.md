### 3. Train v7 — fix wrapper + перезапуск на сервере

**Проблема 1**: `run_with_resume.sh` искал sibling ckpt только на attempt=1 → потеря прогресса при новом запуске.
**Проблема 2**: Sleep 10 сек после HIP crash → следующий attempt падал сразу (VRAM не остыла).
**Проблема 3**: При resume не выкидывал `--continue-from-adapter` → конфликт с `--resume-from-checkpoint`.

**Фикс** (`/home/alex/finetune-env/run_with_resume.sh`):
- `find_latest_sibling_checkpoint()` — ищет ckpt в соседних `<prefix>_*` папках
- Sibling-search **на любом attempt** если local пуст
- Backoff sleep: **30 → 60 → 120 → 240 → 300 сек**
- При resume автоматически удаляет `--continue-from-adapter` из args

## 🚨 Известные грабли

1. **bnb 4-bit + adamw_8bit + `--resume-from-checkpoint`** → `KeyError: 'exp_avg'`. Причина: optimizer.pt из adamw_8bit имеет state в bnb формате (`state1`, `state2`), несовместимо с PyTorch adamw (`exp_avg`). **Решение**: `--continue-from-adapter` (только LoRA веса, optimizer fresh).
2. **adamw_8bit на gfx1201 + r=16/alpha=32** → размер mismatch если ckpt с r=8 (бывало в Phase 5 smoke). **Решение**: использовать **r=8/alpha=16** как в ckpt-180.
3. **GLIBC 2.41 (Debian 13) vs 2.39 (Ubuntu 24)** — но **manylinux2014** wheels (torch ROCm, bnb) совместимы. Бинари llama.cpp требуют пересборки на каждой машине.
4. **HIP headers конфликт**: `libamdhip64-dev 5.7.1` (Ubuntu repo) ↔ `hip-dev 7.2` (repo.radeon.com). Удалить старый dev.
5. **systemd EnvironmentFile + ARGS со spaces** — не работает корректно. Использовать `ExecStart=/bin/bash -c '...llama-server $ARGS'`.

```bash

    --max-retries 6 \
    --output-dir /home/user/finetune-env/output/v7_night_$(date +%H%M) \
    --continue-from-adapter /home/user/finetune-env/output/v7past_1239/checkpoint-180 \
    --model /home/user/offline-debian-pack/1_models/Qwen2.5-Coder-14B-Instruct \
    --dataset /home/user/finetune-env/dataset_v7_train.jsonl \
    --max-seq-len 256 --max-steps 750 --epochs 3 \
    --lora-r 8 --lora-alpha 16 --lora-dropout 0.05 \
    --batch-size 1 --grad-accum 16 --eval-split 0 \
    --save-steps 20 --logging-steps 5 --warmup-steps 10 --lr 1e-4 --seed 42 \
    --bf16 --optim adamw_8bit 2>&1 | stdbuf -oL tr '\r' '\n' | grep --line-buffered -E "loss=|train start|ATTEMPT|FAILED|DONE|memory|✅|❌|🚀|RESUME|illegal|sleep|Найден"
```
