## 🔧 Технические решения и патерны

### 1. Recipe namespace migration (4 фазы)

Для каждого модуля повторено:
- **Phase 1** — replace namespace declarations + using namespace + closing comments + inline FQN через PowerShell regex
- **Phase 2** — `git mv include/<repo>/ include/dsp/<repo>/` + `#include <X/...>` → `<dsp/X/...>`
- **Phase 3** — Doc/ + .rag/ content + переименование `.rag/test_params/{legacy}_ClassName.md` → `dsp_<repo>_ClassName.md` + обновление `.rag/_RAG.md` ссылок
- **Phase B** — structural cleanup: удалить OpenCL `.cl` + `manifest.json`, выпрямить `src/X/src/*.cpp` → `src/X/*.cpp`, обновить `CMakeLists.txt` `target_sources` + комментарий «Стандарт AMD» → «Стандарт DSP-GPU», убрать dead include paths в `python/CMakeLists.txt`

### 2. Особый подход для `drv_gpu_lib` overlap (heterodyne + radar/fm_correlator)

Эти модули определяли классы в `namespace drv_gpu_lib` (legacy GPUWorkLib carry-over). Слепое переименование сломало бы ADL для core классов внутри методов (`IBackend*`, `GpuContext`, `DrvGPU`).

Решение:
```cpp
// Было:
namespace drv_gpu_lib { class HeterodyneDechirp { ... }; }

// Стало:
namespace dsp::heterodyne {
using namespace ::drv_gpu_lib;  // ← импорт core имён
class HeterodyneDechirp { ... };
}
```

`using namespace ::drv_gpu_lib;` после открывающей скобки нового namespace импортирует ВСЕ core имена в `dsp::heterodyne` — методы продолжают резолвить `IBackend*` без префикса. **Антипаттерн в headers, но прагматично.**

### 3. Защита от double-prefix

При FQN replace часто появлялся бактерия `dsp::stats::dsp::stats::` из-за того что regex `\bgpu_sort::` срабатывал на `gpu_sort::` уже после `dsp::stats::`. Решение — добавлен в каждый PowerShell-pass финальный fix:
```powershell
$c2 = $c2 -replace 'dsp::stats::dsp::stats::','dsp::stats::'
```

### 4. Cross-repo cascade

После миграции каждого модуля — обновлены:

## 🎓 Уроки сессии

1. **PowerShell `-replace` boundary boundaries**: `\b` срабатывает после `::`, что приводит к double-prefix при cascade replace. Всегда добавлять fix-pass в конце каждой замены.
2. **Edit без Read падает**: «File has not been read yet» — Read обязателен ДО Edit, даже если файл маленький и понятен.
3. **`using namespace ::drv_gpu_lib;` в headers** — не идеал, но единственный практичный способ мигрировать namespace когда классы используют ADL для core имён. Альтернатива (полные FQN) требует ручного аудита каждого использования.
4. **Изолированность модулей** = огромный бонус. spectrum/stats/signal_generators/etc. не имеют внешних ссылок в коде → миграция намного быстрее чем ожидалось (по 30-60 мин на модуль вместо 2-4ч).
5. **`.rag/test_params/X_ClassName.md` rename** — критично для RAG-индексации, иначе golden_set будет ссылаться на несуществующие файлы.
6. **OpenCL `.cl` файлы найдены в 6 из 7 модулей** в `src/X/kernels/` (legacy) — почистили заодно с миграцией.
7. **Структура `src/X/src/`** (двойная вложенность) — legacy artifact в 5 модулях, выпрямлено в Phase B.
