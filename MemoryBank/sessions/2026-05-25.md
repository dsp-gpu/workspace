## 🎯 Ключевые инсайты

### 1. **14B-DSP не испортил generic skills** ⭐⭐⭐

**Catastrophic forgetting НЕ произошёл!** FT на DSP-GPU dataset (v6) на 10% обучения (375 steps) дал:
- Одинаковое качество на dsp-gpu (3.2) и pao-contrib (3.2)
- Это значит **Plan-D patch (без cast embed/lm_head fp32) + LoRA only on attention** сохраняют generic knowledge модели

**Implication:** наш fine-tune не "сломан" — он просто **не успел выучить факты DSP-GPU** за 10% обучения. Train v7 на 100% должен дать значимый buff на dsp-gpu без потери generic.

### 2. **30B-A3B имеет Ollama repeat-bug**

На pao-contrib T2 (Python review) **зациклилась**: BUG #9-#37 — повторила одно содержимое 28 раз, hit num_predict=4000 limit. Это вытянуло её avg с 4.3 → 3.8.

**Fix:** добавить `repeat_penalty: 1.15` в API request для 30B.

### 3. **35B-Q8 — most stable**

- DSP-GPU: 4.8
- pao-contrib: 4.7
- Δ=0.1 — практически нет разницы

→ **Используем 35B-Q8 для production-grade задач** (deep review, deep documentation, complex describe).

### 4. **Никто не определил pattern HybridBackend = Bridge**

В обоих compare ни одна модель не выдала **Bridge** для `drv_gpu_lib::HybridBackend` (правильный паттерн). Все либо Facade, либо Strategy, либо Composite.

**Это критично подтверждает план v7 fine-tune** — в датасете НУЖНЫ явные `pattern → class` пары (43 пары Bridge+HybridBackend уже в v7).

### 5. **Pimpl pattern определяется правильно** (qjson Parser)

Все 3 модели правильно распознали Pimpl idiom в T3 на qjson. Это значит модели **знают base C++ patterns**, проблема только с domain-specific (HybridBackend).
