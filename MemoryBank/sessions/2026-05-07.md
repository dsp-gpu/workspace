## Вечер 07.05 — диагностический run r=8

**Цель**: понять, r-bottleneck или датасет = причина зацикливания qwen3-8b-dsp в Ollama.

| Параметр | Phase A (утро) | Diagnostic (вечер) |
|----------|----------------|---------------------|
| LoRA r | 4 | **8** |
| seq_len | 384 | 384 |
| epochs | 3 | 3 |
| steps | 411 | 411 |
| runtime | 44 мин | 62 мин (×2 LoRA params) |
| **avg train_loss** | **1.180** | **1.113** (Δ = -0.07) |

**Вывод**: r=8 даёт marginal effect (-0.07 в loss). Обе модели упёрлись в один loss floor ~1.1
→ **bottleneck НЕ в LoRA capacity**, а в **датасете**.

**Главная находка**: предшественница уже подготовила **очищенный датасет**
`C:\finetune-env\dataset_enriched_clean.jsonl` (247 примеров, vs 1093 dirty),
но **ни одна модель ещё не тренировалась на нём**. Это критический эксперимент
на утро 08.05 (~14 мин на 2080 Ti).
