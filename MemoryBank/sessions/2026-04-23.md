## 🛠️ Правки по ходу (минимальные, в scope KernelCache)

### 1. `core/tests/test_storage_services.hpp` — obsolete test out
Удалён вызов `TestKernelCacheService()` (использовал name-only API Phase 1, исчез в Phase A2). Новый `test_kernel_cache_service.hpp` уже покрывает этот функционал 8-ю тестами на `CompileKey`-based API. Старый блок обёрнут в `#if 0 ... #endif` для очистки после Phase E.

### 2. `core/CMakeLists.txt` — rocblas PUBLIC link
```diff
-find_package(hip    REQUIRED)
+find_package(hip     REQUIRED)
+find_package(rocblas REQUIRED)
 find_package(OpenCL REQUIRED)
...
-target_link_libraries(DspCore PUBLIC hip::host OpenCL::OpenCL)
+target_link_libraries(DspCore PUBLIC hip::host OpenCL::OpenCL roc::rocblas)
```
**Причина**: Phase A2 перезаписал `gpu_context.cpp` → теперь зовёт `rocblas_{create,destroy}_handle`, `rocblas_set_stream`. В standalone-сборках линкер проглатывал (as-needed), но при `FetchContent` в `strategies/radar` падала линковка `test_signal_generators_main` / `test_heterodyne_main` на undefined ref. Правка с OK Alex. Модульный подход сохранён: header `gpu_context.hpp` не инклюдит `<rocblas/rocblas.h>`, API остаётся `void* GetRocblasHandleRaw()`.
