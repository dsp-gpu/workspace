### 2. `t_strategies_pipeline.py` (был SKIP `n_fft mismatch (81920) vs (40960)`)
- **Причина 1**: GPU pipeline удваивает FFT-размер (zero-padding ×2 для разрешения). CPU NumpyReference считал FFT длиной только `n_fft`, не `2*n_fft` → shape mismatch.
- **Fix в `DSP/Python/strategies/numpy_reference.py`**:
  - `actual_nfft = 2 * self._n_fft`
  - `padded = np.zeros((n_ant, actual_nfft), ...)`
  - `expected_peak_bin = round(f0 * actual_nfft / fs)`
  - Property `n_fft` → возвращает `2 * self._n_fft`.

### 3. `t_strategies_pipeline.py` (после shape-fix всплыл CHECK-1b_variance edge case)
- **Природа**: для clean CW сигнала `|z| = 1` всегда → `Var(|z|)` теоретически = 0, на float32 ≈ 1.5e-15. И GPU и CPU выдают значения порядка 1e-15, разница ≈ 9e-16. Но `RelativeValidator` использовал hard-coded near-zero порог `1e-15` — variance ровно его перешагивал, метрика становилась относительной → `err/scale = 0.57` → FAIL.
- **Fix в `DSP/Python/common/validators/numeric.py`**:
  - Добавлен параметр `abs_threshold: float = 1e-4` в `RelativeValidator.__init__`.
  - Near-zero ветка теперь использует `self._abs_tol` для **обоих** условий: `scale < self._abs_tol` (детект) и `passed = err < self._abs_tol` (порог).
  - По умолчанию 1e-4 (float32-friendly); параметр настраиваемый при создании валидатора.
- Результат: все 4 теста v1-v4 → 32/32 PASS (v5 — справедливый SKIP, нет файла).
