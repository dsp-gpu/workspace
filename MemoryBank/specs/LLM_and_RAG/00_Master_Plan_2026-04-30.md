## 1. Что строим

Локальный AI-ассистент **`dsp-asst`** для проекта **DSP-GPU** (10 репо C++/HIP/Python).

**Способности:**
- Ориентируется в коде: классы, функции, public-переменные, namespace, граф зависимостей.
- Отвечает на вопросы по проекту с цитатой из исходника (RAG).
- Пишет тесты в стиле `common.runner.TestRunner` (граничные значения — из базы `test_params`).
- Пишет код в стиле проекта: ООП, SOLID, GRASP, GoF, паттерны уже использующиеся (Bridge, Factory, Strategy, RAII, Facade…).
- Пишет doxygen-комменты → можно собрать LaTeX через Doxygen toolchain.
- **Режим агента**: сам ходит в Context7 / WebFetch / sequential-thinking / GitHub при сложных задачах.

**Модели и окружения (по горизонту):**

| Этап | Окружение | Модель | LLM-runtime | Когда |
|------|-----------|--------|-------------|-------|
| 1 | Дома (Windows) | Qwen3 8B Q4_K_M | Ollama | сейчас, отладка |
| 2 | Работа (Debian, твой ПК) | Qwen3 32B Q4 | Ollama | после отладки 8B |
| 3 | Мини-сервер (Ubuntu, копия 2.2) | Qwen3 32B AWQ | vLLM (systemd) | пробное многопользовательское |
| 4 | Production (общий сервер / A100) | DeepSeek-V3 / Qwen Max | vLLM | +3-4 мес |

> Все решения должны **переноситься** между этапами без переписывания индекса/промптов. Меняется только runtime-LLM.

**Ключевые принципы:**

1. **Один сервис — много клиентов.** Continue, Claude Code, CLI — все ходят в один Python-сервис.
2. **MCP — главная точка входа.** Continue (с осени 2025) и Claude Code (Кодо) умеют MCP.
3. **Индекс на каждой машине свой.** Не синхронизируем — каждая машина строит за ~10 мин.
4. **Code-first метаданные.** Из архитектуры берём: repo, namespace, layer, pattern, dep-graph. Это богатые фильтры для retrieval.
5. **Инкрементальное переиндексирование.** По file-hash (blake3): меняешь 5 файлов — обновляются 5 чанков.
6. **Symbol-graph как код первого класса.** PostgreSQL хранит классы/методы/поля + 16 типов зависимостей (call/inherit/include/uses_type/...).
7. **База граничных значений (`test_params`).** AI извлекает из кода → оператор правит JSON в VSCode → промпт «напиши тест» получает её как контекст.

## 3. Режимы работы

| Режим | Что делает | Retrieval-коллекция | Промпт |
|-------|------------|---------------------|--------|
| `explain` | Объясняет класс/функцию/файл из проекта | `public_api` + symbol-graph соседи | `prompts/explain_*` |
| `find` | «Где вызывается X», «кто наследует Y» | `public_api` + symbol-graph + `pg_trgm` | `prompts/find_*` |
| `test` | Пишет тест (Python через TestRunner / C++ functional / C++ benchmark) | `public_api` + `test_params` | `prompts/002`, `003`, `004` |
| `doxy` | Добавляет/правит doxygen-комменты в файле | `public_api` + соседи по include | `prompts/doxy_*` |
| `refactor` | Рефакторит под SOLID/GRASP/GoF, сохраняет API | `internal` (включая private) | `prompts/refactor_*` |
| **`agent`** | **Сложная задача**: сам декомпозирует, ходит в Context7/Web | `internal` динамически | `prompts/agent_*` |

> Промпты лежат в `prompts/NNN_*.md`. Локальная модель (Qwen 8B) **глупее меня** — промпты должны быть короткие и шаблонные. См. `prompts/README.md`.

> **C++ тесты в проекте — три вида:**
> - **A. Functional** (`test_*.hpp`): свободная `inline void run()` + C++ `TestRunner runner` + lambda-кейсы. Промпт `003`.
> - **B. Benchmark** (`*_benchmark_*.hpp`): наследует `drv_gpu_lib::GpuBenchmarkBase`, владеет ссылкой на тестируемый класс, профилирование через `ProfilingFacade::BatchRecord`. Промпт `004`.
> - **C. Python wrapper** (`t_*.py`): через `common.runner.TestRunner`. Промпт `002`.

## 7. Финальные решения Итерации 2 (зафиксировано)

| Вопрос | Решение |
|--------|---------|
| Имя ассистенту | **`dsp-asst`** (CLI + MCP-имя), пакет = `dsp_assistant` |
| Окружения | 4 этапа: home Win → work Debian → мини-сервер Ubuntu → prod (общий сервер) |
| Где запускается MCP | На каждой dev-машине локально (1-2). На мини-сервере (3) — через SSH-туннель к dev-машинам |

## 8. Что НЕ делаем (anti-scope)

- ❌ Не пишем свой web-UI для чата — Continue/Claude Code достаточно.
- ❌ Не индексируем `E:\C++\GPUWorkLib\` (legacy, путает API).

- ❌ Не используем SQLite — везде PostgreSQL.
