# Выбор модели роутера llm-router — глубокий анализ

> **Вопрос Alex (2026-06-08):** поставить ли в роутер **Qwen3 1.5B обучаемый** (генеративная LLM)
> вместо MLP-головы из спеки? Есть ли speed-модификации?
> **Метод:** research статей/github (RouteLLM, RAGRouter, SetFit, ModernBERT, SALSA) + сверка с локальным парком.
> **Связь:** `llm_router_architecture_2026-06-07.md` (§ trained head Phase 3), `TASK_llm_router_phase0.md`.

---

## 🎯 Короткий ответ (TL;DR)

**Qwen3-1.5B (генеративную LLM) в роутер — НЕ ставить.** Для задачи «1 запрос → 1 из 3 классов
{A, B, CLARIFY}» на 10-20K примеров генеративная LLM **проигрывает по всем осям**: лишние
1.1-1.8 ГБ VRAM, на 1-2 порядка больше latency (автогенерация vs один forward), и **НЕ точнее**
лёгкой головы. План спеки (**MLP-голова на bge-m3, 0 ГБ VRAM**) — архитектурно верный и
совпадает с тем, что использует сам RouteLLM (их лучший роутер — НЕ LLM).

> ⚠️ Важный факт: **«Qwen3 1.5B» не существует.** Линейка Qwen3 dense: **0.6B** и **1.7B**
> (между ними 1.5B нет). Дальше считаю по реальным 0.6B / 1.7B.

## 📊 Сравнение подходов (на наш кейс: 3 класса, 10-20K, bge-m3 уже считаем)

| Подход | Точность | VRAM доп. | Latency/запрос | Обучение | Данные |
|--------|----------|-----------|----------------|----------|--------|
| **MLP-голова на bge-m3** ⭐ (1024 + фичи → MLP → 3) | высокая (closed-set) | **0 ГБ** (эмбеддинг уже есть) | **~мс** (1 matmul) | минуты (torch/sklearn) | 1-20K |
| **ModernBERT-base** (план Б) | SOTA closed-set (F1 0.93 на 77 кл.) | ~0.3-0.6 ГБ | единицы мс (~277 sample/s) | ~5 мин/5 эпох | 10-20K идеал |
| **SetFit** (план В при малой разметке) | ≈ fine-tuned энкодер | ~0 (та же sent-модель) | мс-десятки | 30 сек few-shot | **8-55 примеров/класс** |
| Qwen3-1.7B generative | НЕ выше энкодера | **1.1-1.8 ГБ** | десятки-сотни мс (автогенерация) | QLoRA, сложно | нужно больше |
| Qwen3-1.7B logit-class (SALSA) | ≈ энкодер | 1.1-1.8 ГБ | 1 forward (но > MLP) | PEFT + class-token | среднее |

## 🔬 Доказательная база (что нашли)

1. **RouteLLM (LMSYS, arXiv 2406.18665, github lm-sys/RouteLLM).** 4 роутера; **победитель —
   matrix factorization (НЕ LLM)**: 95% качества GPT-4 при 14-26% вызовов дорогой модели.
   Overhead роутинга ≤ 0.4% стоимости генерации. Их топовые роутеры = лёгкие головы на
   эмбеддингах — **ровно наш вариант MLP**.
2. **RAGRouter (arXiv 2505.23052)** — даже SOTA-роутер это **энкодер ~136M** (mpnet +
   cross-encoder + обучаемые capability-векторы), а не генеративная LLM. 64.46% acc (+3.6 пп).
3. **Encoder ≈ 20× быстрее малой LLM** (277 vs 12 sample/s) при сопоставимой/выше точности на
   closed-set (alex-jacobs «Beating BERT?»).
4. **SetFit (arXiv 2209.11055)** — 0.9 acc от 55 примеров; 8 примеров/класс бьют fine-tuned
   трансформеры (×27 меньше данных).
5. **SALSA (arXiv 2510.22691)** — если уж decoder: class→token + проекция логитов, **1 forward**,
   кратно быстрее генерации. Но для 3 классов сложнее MLP без выигрыша.

---

## 🧠 Почему MLP-голова, а не Qwen3 (по сути)

- **Эмбеддинг bge-m3 мы и так считаем** для RAG `/search`. MLP-голова **переиспользует** его →
  0 ГБ, 0 лишних загрузок модели. Генеративная LLM — это отдельные 1.1-1.8 ГБ + свой forward.
- **Классификация ≠ генерация.** «1 из 3» — это softmax по одному вектору. Автогенерация
  (даже constrained/speculative) по определению медленнее одного forward pass.
- **GRASP/«не плодить сущности» (правило Alex):** роутер-голова живёт внутри сервиса, без
  второго llama-server, без второй модели в VRAM (а у нас Tract A 14B + Tract B 35B уже едят GPU).
- **Точность.** На closed-set 10-20K малая LLM НЕ обгоняет энкодер/голову — выигрыш LLM только
  в open-set/zero-shot/рассуждениях, чего у роутинга «наше/не наше/уточнить» нет.

## ✅ Рекомендация (план действий)

1. **Phase 3 = MLP-голова на bge-m3** (как в спеке §11: `[2061 → 256 → 64 → 3]`). Оставляем.
   Обучение — минуты на 10-20K + production-логи. 0 ГБ VRAM.
2. **CLARIFY — через порог уверенности**, а не отдельным «трудным» классом: `max-softmax < τ` →
   CLARIFY (τ калибруем на валидации). Робастнее на дисбалансе, чем учить «неопределённость».
   (Явный 3-й класс можно оставить, но порог надёжнее на границе A/B.)
3. **План Б (если точности MLP мало):** fine-tune **`answerdotai/ModernBERT-base`** (~0.15 ГБ
   веса, ~0.3-0.6 ГБ в трейне, F1 0.9+). Отдельные веса, но дёшево.
4. **План В (мало разметки):** **SetFit** поверх bge-m3 — 8-55 примеров/класс.
5. **Qwen3-1.7B/0.6B generative — НЕ берём** в роутер. Если когда-нибудь понадобится decoder —
   только SALSA-стиль (logit-classification, 1 forward), не генерация.
