# `.future/` — Перспективные задачи DSP-GPU

> **Назначение**: задачи, которые **не запланированы** в текущий roadmap, но **должны быть** когда-то реализованы. Идеи и фичи, отложенные осознанно (не «забытые»).

## Чем отличается от других папок

| Папка | Когда использовать |
|-------|-------------------|
| `tasks/` | Активные задачи (in progress / next sprint) |
| `archive/` | Завершённые задачи (для истории) |
| `specs/` | Спецификации / исследования / ревью |
| **`.future/`** | **Перспективные** идеи / фичи / рефакторинги — НЕ в работе, НО зафиксированы |

## Формат файла

`TASK_<topic>_<scope>.md` — без даты (это идея, не план):

```markdown
# TASK: <Название> (perspective)

**Создано**: YYYY-MM-DD
**Статус**: 📌 perspective (не запланировано)
**Триггер реактивации**: <условие, когда взять в работу>

## Зачем
<контекст / мотивация>

## Что нужно сделать
<высокоуровневые шаги>

## Зависимости
<что должно быть готово перед стартом>

## Связанные документы
<ссылки на спеки / тесты / коммиты>
```

## Workflow

1. **Создание**: когда обнаруживается «надо бы сделать X, но не сейчас» → файл сюда.
2. **Хранение**: тут же, читается **редко** (не в каждой сессии).
3. **Реактивация**: когда триггер сработал → переместить в `tasks/TASK_<topic>_<phase>.md` + добавить в `IN_PROGRESS.md`.

## Что НЕ кладём

- ❌ Текущие задачи (→ `tasks/`)
- ❌ Завершённые (→ `archive/`)
- ❌ Идеи на «может быть» / brainstorm (→ `specs/`, как research-документ)
- ❌ Bugs (→ обычные `tasks/`, потому что bug требует решения)
