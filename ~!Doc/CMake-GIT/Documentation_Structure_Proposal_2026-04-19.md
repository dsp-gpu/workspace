## 🧭 Принципы декомпозиции

| Принцип | Почему так |
|---------|-----------|
| **Один файл = один вопрос** | Читатель пришёл с вопросом «как перенести на флешку» — открыл `07_Transfer_To_Offline_PC.md`, не листает 1000 строк |
| **README как карта** | ≤150 строк, в 30 секунд понимаешь куда идти |
| **Нумерация 00-12** | Естественный порядок чтения для полного onboarding; при этом можно дёргать любой файл отдельно |
| **Размер каждого 200-500 строк** | Чтение за 10-15 минут, целостная картина без усталости |
| **Cross-links везде** | Каждый термин линкует в глоссарий, каждый скрипт — в `09_Scripts_Reference.md` |
| **Большой guide не исчезает** | `Distributed_Modules_Guide_2026-04-19.md` становится **архивным** (или удаляется после split). Его содержимое расходится по 01-10 |

## 📄 Учёт будущей конверсии в docx / pdf / LaTeX

Alex сказал: «потом сделаем в разных форматах — docx, pdf, LaTeX». Это влияет на то, как писать markdown **уже сейчас**, чтобы pandoc-конверсия прошла без боли.

### Правила стиля с этого момента

| Деталь | Что делаю | Почему |
|--------|-----------|--------|
| **Диаграммы** | Дублирую: ASCII + `mermaid`/`plantuml` блок | ASCII сломается в pdf (шрифты); mermaid pandoc рендерит в svg |
| **Эмодзи** | Использую умеренно, всегда рядом с текстовой подписью (`✅ Готово`, не просто `✅`) | В некоторых pdf-рендерах эмодзи = tofu-квадратики |
| **Таблицы** | Только стандартный GFM markdown (без HTML) | pandoc ест чисто |
| **Cross-links** | Относительные + якоря `[текст](01_Zones.md#zone-0)` | После `pandoc --combine` работают |
| **Заголовки** | Без уникальных символов в тексте заголовков (чистые ASCII + эмодзи в начале) | Стабильные anchors для ToC |
| **Код-блоки** | С явным языком ` ```bash `, ` ```cmake `, ` ```json ` | pandoc применит syntax highlighting |
| **Списки** | Стандартные `-` / `1.` без вложенности глубже 3 уровней | В docx глубже ломается |

### Команды конверсии (потом, когда понадобится)

```bash
# PDF через LaTeX
pandoc README.md 00_Glossary.md 01_*.md 02_*.md ... \
       --toc --number-sections --pdf-engine=xelatex \
       -o Distributed_Modules_Guide.pdf

# Word
pandoc README.md 00_*.md 01_*.md ... \
       --toc --number-sections \
       -o Distributed_Modules_Guide.docx

# LaTeX исходник (для дальнейшей ручной доводки)
pandoc README.md 00_*.md ... \
       --toc --number-sections \
       -o Distributed_Modules_Guide.tex
```

Декомпозиция на 13 файлов для pandoc **удобнее**, чем один огромный: конкатенирует в нужном порядке + ToC автоматически.
